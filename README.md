# ShopGen

ShopGen — это веб-приложение для автоматической генерации описаний товаров для интернет-магазина.

Проект помогает быстро создавать текстовые описания для карточек товаров на основе их характеристик. Пользователь выбирает категорию товара, заполняет поля формы и получает готовое описание, которое можно использовать в интернет-магазине или доработать вручную.

## О проекте

В интернет-магазинах часто нужно создавать много похожих карточек товаров. Если писать описание для каждого товара вручную, это занимает много времени, особенно при большом каталоге. ShopGen автоматизирует этот процесс: сервис берет введенные характеристики товара и на их основе формирует связный рекламный текст.

Генерация построена на шаблонном подходе. Для каждой категории товаров подготовлены свои шаблоны и наборы фраз. Благодаря этому результат получается быстрым, предсказуемым и понятным. При этом описания могут отличаться друг от друга, потому что используются разные варианты фраз и шаблонов.

Сейчас приложение поддерживает несколько категорий товаров: смартфоны, кроссовки и видеокарты. Для каждой категории отображаются свои поля. Например, для смартфона можно указать бренд, модель, диагональ экрана, объем оперативной памяти, встроенную память, батарею, камеру и ключевую особенность. Для кроссовок и видеокарт используются другие характеристики, подходящие именно этим товарам.

После заполнения формы пользователь нажимает кнопку генерации. Данные отправляются на сервер, где выбирается подходящий шаблон, подставляются характеристики товара и добавляются дополнительные фразы. Готовое описание возвращается на страницу и сохраняется в историю. Историю можно просматривать и фильтровать по категориям.

## Возможности

- генерация описаний по характеристикам товара;
- поддержка нескольких категорий товаров;
- автоматическая смена полей формы при выборе категории;
- вариативность текста за счет шаблонов и словарей фраз;
- использование seed для повторяемого результата;
- сохранение истории генераций;
- фильтрация истории по категориям;
- копирование готового описания;
- очистка формы;
- работа через веб-интерфейс;
- работа через REST API;
- возможность обработки одного товара или списка товаров.

## Используемые технологии

В проекте используются Python, Flask, SQLite, HTML, и Render.

Backend отвечает за обработку запросов, генерацию описаний и сохранение истории. Frontend отвечает за форму ввода, выбор категории, отображение результата и историю генераций. Для хранения данных используется SQLite. Проект развернут на Render и доступен по публичной ссылке.

## Как запустить проект локально

Сначала нужно склонировать репозиторий и перейти в папку проекта:

```bash
git clone https://github.com/kosavka223/ShopGen
cd ShopGen
```

Затем создать виртуальное окружение:

```bash
python -m venv venv
```

Активировать виртуальное окружение.

Для macOS или Linux:

```bash
source venv/bin/activate
```

Для Windows:

```bash
venv\Scripts\activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Запустить приложение:

```bash
python app.py
```

После запуска сайт будет доступен в браузере:

```text
http://127.0.0.1:5000
```

Проверить работу сервера можно по адресу:

```text
http://127.0.0.1:5000/ping
```

Если сервер работает, он вернет ответ со статусом `ok`.

## API

Основной маршрут для генерации описания:

```text
POST /api/generate
```

Пример запроса для смартфона:

```json
{
  "category": "smartphone",
  "brand": "Samsung",
  "model": "Galaxy A55",
  "screen_size_in": 6.6,
  "ram_gb": 8,
  "storage_gb": 256,
  "battery_mah": 5000,
  "camera_mp": 50,
  "key_feature": "5G и быстрая зарядка"
}
```

Пример ответа:

```json
{
  "success": true,
  "count": 1,
  "results": [
    {
      "category": "smartphone",
      "description": "Удобный смартфон Samsung Galaxy A55 — для учебы и развлечений. Экран 6.6 дюйма удобен для контента, а 8 ГБ RAM и 256 ГБ памяти подходят для приложений и файлов. Аккумулятор 5000 мА·ч помогает дольше оставаться на связи. Дополнительное преимущество — 5G и быстрая зарядка.",
      "saved_to_history": true
    }
  ]
}
```

Историю генераций можно получить через:

```text
GET /api/history
```

Историю по конкретной категории:

```text
GET /api/history?category=smartphone
```

Ограничение количества записей:

```text
GET /api/history?limit=10
```

Получение одной записи по ID:

```text
GET /api/history/1
```

## Пример работы

Пользователь выбирает категорию «Смартфон» и вводит данные:

```text
Бренд: Samsung
Модель: Galaxy A55
Экран: 6.6
RAM: 8 ГБ
Память: 256 ГБ
Батарея: 5000 мА·ч
Камера: 50 Мп
Особенность: 5G и быстрая зарядка
```

Система генерирует описание:

```text
Удобный смартфон Samsung Galaxy A55 — для учебы и развлечений. Экран 6.6 дюйма удобен для контента, а 8 ГБ RAM и 256 ГБ памяти подходят для приложений и файлов. Аккумулятор 5000 мА·ч помогает дольше оставаться на связи. Дополнительное преимущество — 5G и быстрая зарядка.
```

## Добавление новой категории

Проект можно расширять новыми категориями товаров. Для этого нужно добавить шаблоны описаний, фразы для вариативности, обработку характеристик на backend и поля формы на frontend.

Например, если нужно добавить категорию ноутбуков, сначала создаются шаблоны описаний с переменными:

```json
"laptop": [
  "{adj} {title}. {processor_phrase}{memory_phrase}{screen_phrase}{feature_phrase}{cta}",
  "{title}. {screen_phrase}{processor_phrase}{memory_phrase}{feature_phrase}{cta}"
]
```

Затем добавляются фразы, которые будут использоваться при генерации:

```json
"laptop_use_case": [
  "Подходит для учебы и работы. ",
  "Хороший вариант для повседневных задач. ",
  "Подойдет для офиса, учебы и мультимедиа. "
]
```

После этого на backend добавляется логика, которая получает характеристики ноутбука и формирует части текста: название товара, фразу про процессор, память, экран и особенности. На frontend добавляются поля для ввода характеристик: бренд, модель, процессор, диагональ экрана, оперативная память, встроенная память и ключевая особенность.

После добавления этих частей новая категория становится доступна в интерфейсе и через API.

## Возможности для развития

В дальнейшем проект можно расширить: добавить больше категорий товаров, сохранять финальную версию описания, добавить массовую загрузку товаров из CSV или Excel, сделать экспорт результатов, улучшить правила согласования слов и добавить разные стили описания: краткий, рекламный или технический.
